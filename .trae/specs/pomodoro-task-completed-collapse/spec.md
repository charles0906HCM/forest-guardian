# 番茄钟任务列表已完成任务折叠 - 产品需求文档

## Overview
- **Summary**: 番茄钟页面的任务列表默认只显示未完成的任务，已完成的任务默认折叠隐藏，通过按钮可以切换显示/隐藏已完成任务
- **Purpose**: 减少视觉干扰，让用户更专注于当前待完成的任务，同时保留查看已完成任务的能力
- **Target Users**: 使用番茄专注功能的所有用户

## Goals
- 默认只显示未完成的任务
- 已完成的任务默认折叠隐藏
- 添加按钮切换显示/隐藏已完成任务
- 按钮显示已完成任务的数量
- 折叠/展开状态有平滑的动画效果

## Non-Goals (Out of Scope)
- 不改变任务完成状态的切换逻辑
- 不改变任务选择逻辑
- 不改变任务列表的样式设计

## Background & Context
- 当前番茄钟页面左侧显示所有任务（包括已完成和未完成）
- 当用户完成多个任务后，已完成任务会占据大量空间，影响未完成任务的可见性
- 用户希望更清晰地看到待完成的任务

## Functional Requirements
- **FR-1**: 默认只显示未完成的任务
- **FR-2**: 已完成任务默认折叠隐藏
- **FR-3**: 添加"显示已完成任务"按钮，点击后展开已完成任务
- **FR-4**: 按钮显示已完成任务的数量（如"显示 3 个已完成任务"）
- **FR-5**: 展开后按钮文字变为"隐藏已完成任务"
- **FR-6**: 折叠/展开时有平滑的过渡动画

## Non-Functional Requirements
- **NFR-1**: 折叠/展开动画应在 200-300ms 内完成
- **NFR-2**: 按钮样式应与现有设计风格一致
- **NFR-3**: 展开的已完成任务应显示为灰色/删除线样式，与未完成任务区分

## Constraints
- **Technical**: 使用 React useState 控制折叠状态
- **Dependencies**: 依赖现有的任务数据和完成状态判断逻辑

## Assumptions
- 任务完成状态由 isTaskCompletedOnDate 函数判断
- 任务列表组件结构保持不变

## Acceptance Criteria

### AC-1: 默认只显示未完成任务
- **Given**: 用户打开番茄钟页面
- **When**: 页面加载完成
- **Then**: 任务列表只显示未完成的任务，已完成任务不可见
- **Verification**: `human-judgment`

### AC-2: 显示已完成任务按钮
- **Given**: 页面上有已完成的任务
- **When**: 用户查看任务列表
- **Then**: 在未完成任务下方显示"显示 X 个已完成任务"按钮（X 为已完成任务数量）
- **Verification**: `human-judgment`

### AC-3: 点击按钮展开已完成任务
- **Given**: 已完成任务处于折叠状态
- **When**: 用户点击"显示 X 个已完成任务"按钮
- **Then**: 已完成任务展开显示，按钮文字变为"隐藏已完成任务"
- **Verification**: `human-judgment`

### AC-4: 点击按钮折叠已完成任务
- **Given**: 已完成任务处于展开状态
- **When**: 用户点击"隐藏已完成任务"按钮
- **Then**: 已完成任务折叠隐藏，按钮文字变为"显示 X 个已完成任务"
- **Verification**: `human-judgment`

### AC-5: 无已完成任务时不显示按钮
- **Given**: 页面上没有已完成的任务
- **When**: 用户查看任务列表
- **Then**: 不显示"显示已完成任务"按钮
- **Verification**: `human-judgment`

### AC-6: 折叠/展开有动画效果
- **Given**: 用户点击折叠/展开按钮
- **When**: 状态切换时
- **Then**: 已完成任务区域有平滑的高度过渡动画
- **Verification**: `human-judgment`
